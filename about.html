Bio

Hello, I'm Dinesh.

I graduated with a BE in Computer Science Engineering, expecting to spend my career writing backend logic. Instead, a college project — redesigning a clunky internal portal — pulled me toward interface design, and I never really left.

What keeps me there is the combination: I can sketch a flow in Figma, argue for it with research and reasoning, and then sit down and build it myself in HTML, CSS and JavaScript. Nothing gets lost in translation between "designed" and "shipped" because I do both.

Outside of pure UI work, I'm genuinely curious about the business side — digital marketing, conversion, and using SQL and Excel to turn raw data into a dashboard someone can actually act on. I like knowing whether a design decision actually moved a number.

When I'm not designing, I'm usually breaking down an app I use daily to understand why one particular screen feels effortless and another doesn't.

How I work

Three habits behind every project.

Research first

Every redesign starts by using the existing product like a real user, listing friction points before opening Figma.

Design in systems

Reusable components, consistent spacing and type scales — so a design scales past one screen.

Build it myself

I code my own designs whenever I can, because that's where you learn what actually works in a browser.
